## 2) 상세 내용

### 핵심 아이디어: 7B 오픈소스로 55B 클로즈드 모델을 능가

RT-2는 웹 스케일 VLM이 로봇 제어에 전용될 수 있음을 입증했지만, 가중치 비공개·55B 파라미터 비용·학습 데이터 미공개라는 한계가 있었다. OpenVLA의 질문: *비용의 일부로 RT-2에 필적하거나 능가하는 오픈소스 모델을 만들 수 있는가?*

OpenVLA는 **Prismatic VLM** 백본(SigLIP + DINOv2 비전 인코더 + Llama 2 7B) 위에 구축된 완전 오픈소스 7B VLA 모델이다. **Open X-Embodiment(OXE)** 데이터셋(22개 로봇 환경, 527가지 스킬, 97만 개 궤적)으로 학습한다.

### 아키텍처

OpenVLA는 **Prismatic VLM** 아키텍처를 기반으로 한다:

- **비전 인코더 (듀얼):** SigLIP (언어-이미지 사전학습) + DINOv2 (자기지도 ViT); 두 특징을 이어붙여 더 풍부한 시각 표현 제공
- **프로젝션:** 이어붙인 시각 특징을 LLM 토큰 공간으로 매핑하는 선형 레이어
- **언어 모델:** Llama 2 7B (디코더 전용); 시각 토큰 + 텍스트 토큰을 함께 처리
- **액션 예측:** 일반 텍스트 예측과 동일하게 액션 토큰을 자기회귀적으로 예측

### 액션 표현

로봇 액션은 **7차원 벡터**로 표현된다:

- 6-DoF 엔드이펙터 델타 (Δx, Δy, Δz, Δroll, Δpitch, Δyaw)
- 그리퍼 명령 1개 (열기/닫기)

7개 차원 각각을 **256 bins으로 이산화**해 LLM 어휘의 텍스트 토큰으로 매핑한다. 모델은 7개 토큰을 순서대로 예측한다 — RT-2와 동일한 토큰 기반 방식이지만 완전히 오픈소스다.

### 학습 데이터: Open X-Embodiment (OXE)

- **97만 개 궤적**, 22개 로봇 환경
- 다양한 환경에 걸친 527가지 스킬
- 원격 조작 데모, 스크립트 정책, 인간 영상 혼합
- 공개 데이터 → 학습 재현 가능

OpenVLA는 환경과 스킬 유형 간 균형을 위해 정밀하게 조정된 **데이터 믹스처**를 사용한다 — 희귀 스킬은 오버샘플링, 과잉 대표된 환경은 다운샘플링.

### 결과

**BridgeV2 벤치마크** (실제 로봇, 미관측 물체):

| 모델 | 파라미터 | 성공률 |
|------|---------|-------|
| OpenVLA | 7B | **73.7%** |
| RT-2-X | 55B | 67.3% |
| Octo | 93M | 56.9% |

**파인튜닝:** LoRA를 사용하면 단 ~100개 데모로 새 로봇·태스크에 적응 가능.

### 한계

- **추론 속도:** 단일 GPU에서 7B 모델 구동은 소형 특화 모델보다 느려 고주파 제어(>5 Hz)에는 아직 부적합
- **단일팔 한정:** OXE는 단일팔 조작이 대부분; 바이매뉴얼·모바일 조작 일반화는 미검증
- **시간 맥락 없음:** 스텝당 단일 이미지 프레임만 입력, 명시적 히스토리 인코딩 없음

## 3) 왜 중요한 논문인가

- **7B 오픈소스 모델이 55B 클로즈드 모델을 능가**할 수 있음을 입증해, 'VLA 성능은 규모에 달려있다'는 통념을 깼다.
- **Open X-Embodiment를 실용적인 학습 기반**으로 확립하고, 데이터 양이 아닌 **데이터 믹스처 전략**이 성능을 좌우함을 보여준다.
- 가중치·코드·학습 데이터가 모두 공개돼 완전히 재현 가능한 기준 모델을 제공, 커뮤니티가 VLA 동작을 상세히 연구하고 발전시킬 수 있게 했다.
- ~100개 데모로 새 로봇에 LoRA 파인튜닝 가능 → Config의 배포 시나리오에서 **퓨샷 적응**의 실용적 경로를 제시한다.

## 4) Config 적용 사례

- **오픈소스를 출발점으로:** OpenVLA-7B는 Config 로보틱스 작업의 실용적인 기반 모델이다. LoRA로 Config 로봇과 태스크에 파인튜닝하는 것이 처음부터 학습하는 것보다 비용이 훨씬 적다.
- **데이터 믹스처 전략:** 희귀 스킬 오버샘플링, 과잉 대표 환경 다운샘플링이라는 OXE 믹스처 레시피는 Config가 자체 멀티로봇 학습 데이터를 구성할 때 바로 활용할 수 있다.
- **듀얼 비전 인코더 설계:** SigLIP(언어 정렬)과 DINOv2(공간·구조적) 인코더를 함께 쓰면 상호 보완적인 시각 특징을 제공한다. Config는 이 구성이 자사 조작 태스크의 그라운딩을 개선하는지 평가할 수 있다.
- **벤치마크 설계:** 미관측 물체로 BridgeV2를 평가하는 방식은 Config 자체 평가 하네스의 구체적인 템플릿이다 — 구조적이고 재현 가능하며 in-distribution이 아닌 일반화를 측정한다.
- **추론 파이프라인:** 논문의 처리량 분석(단일 GPU, 최적화 시 ~6 Hz)은 Config의 온로봇 배포 계획에서 현실적인 예산 기준을 제공한다.
